Explicacion interactiva: mini juego 2D en Java

Esta guia recorre el codigo en orden de ejecucion: UI, motor de juego, logica y tests.

1) Arquitectura del proyecto

UI

App.java

Crea la ventana Swing, captura teclado y pinta en pantalla.

Motor

GameEngine.java

Actualiza estado por frame: movimiento, colisiones, puntos y reset.

Reglas puras

GameLogic.java

Funciones sin UI para limites, rebotes, colision y velocidad aleatoria.

Calidad

Tests

Unit tests para reglas e integration tests para ciclo completo de juego.

2) Paso a paso del flujo

Arranque de la app main -> ventana
App.main ejecuta SwingUtilities.invokeLater, crea JFrame, agrega GamePanel y muestra la ventana.
SwingUtilities.invokeLater(() -> { JFrame frame = new JFrame("Mini Juego 2D - Esquiva Cuadrados"); frame.setContentPane(new GamePanel()); frame.pack(); frame.setVisible(true); });
Bucle de juego Timer cada ~16ms
El Timer llama updateGame() y repaint() en cada tick. Eso produce ~60 FPS.
this.gameTimer = new Timer(16, event -> { updateGame(); repaint(); });
Actualizacion de estado GameEngine.tick
Tick aplica teclado, limita al jugador, mueve enemigo, suma puntos si sale de pantalla y verifica colision.
engine.tick(leftPressed, rightPressed, upPressed, downPressed);
Renderizado paintComponent
Dibuja fondo, jugador, enemigo, HUD de puntos y cartel de game over cuando running = false.
Reinicio tecla R
Si el juego termino, al pulsar R se restaura score, posiciones y estado running.

4) Como leer los tests

Unit tests (GameLogic)
Verifican funciones puras: limites, rebote, colision, rango de velocidad.
Integration tests (GameEngine)
Verifican el flujo de varios pasos: movimiento por frame, suma de puntos al salir y stop al colisionar.
Comando de ejecucion
Ejecuta pruebas con Maven:
mvn test
Ejecuta el juego:
mvn exec:java
Sugerencia: manten App como UI, GameEngine como estado y GameLogic como reglas puras para escalar el juego sin romper tests.
